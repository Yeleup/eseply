# Модуль: Начисления

## Терминология

Начисление в коде называется `Accrual`.

Таблица начислений называется `accruals`.

## Назначение

Модуль хранит результат закрытия расчётного месяца по абоненту.

Сейчас реализован расчёт по фиксированной сумме, на одного человека и по счётчику.

После успешного создания начисления закрытие месяца создаёт квитанцию `Receipt` из сохранённого `Accrual`.

## Основные поля

| Поле | Описание |
|---|---|
| Организация | Tenant-организация, поле `organization_id` |
| Абонент | Клиент, поле `client_id` |
| Услуга | Услуга выбранной tenant-организации, поле `utility_service_id` если хранится технически |
| Расчётный месяц | Связь с `BillingPeriod`, поле `billing_period_id` |
| Лицевой счёт | Снимок лицевого счёта, поле `account_number` |
| Абонент | Снимок имени / наименования клиента, поле `client_name` |
| Название услуги | Снимок названия услуги, поле `utility_service_name` |
| Тип начисления | Тип начисления клиента на момент закрытия, поле `billing_type` |
| Объём | Расход или количество проживающих, поле `volume` |
| Тариф | Использованная цена, поле `tariff_price` |
| Начислено | Сумма начисления, поле `amount` |
| Оплачено | Сумма оплат за период, поле `paid_amount` |
| Корректировка | Сумма корректировок сальдо за период, поле `adjustment_amount` |
| Начальное сальдо | Сальдо на начало периода, поле `opening_balance` |
| Конечное сальдо | Сальдо на конец периода, поле `closing_balance` |
| Дата закрытия | Когда выполнено закрытие, поле `closed_at` |

## Правила расчёта

Закрытие месяца рассчитывает всех активных абонентов.

Коммунальная услуга берётся из выбранной tenant-организации.

У абонента не выбирается отдельная услуга.

Для расчёта на одного человека у абонента должен быть выбран тип абонента.

Для расчёта на одного человека должен существовать активный `Tariff` для типа абонента с заполненным `per_person_price`.

Для расчёта по счётчику у абонента должен быть выбран тип абонента.

Для расчёта по счётчику должны существовать один или несколько активных счётчиков по услуге организации, показание за период по каждому активному счётчику и активный `Tariff` для типа абонента с заполненным `unit_price`.

Формула фиксированного начисления:

`Начисление = fixed_amount`

Формула начисления на одного человека:

`Начисление = residents_count * per_person_price`

Формула начисления по счётчику:

`Начисление = сумма consumption по всем активным счётчикам абонента * unit_price`

Поле `volume` хранит расход по счётчику или количество проживающих для начисления на одного человека.

Поле `tariff_price` хранит цену тарифа, использованную при расчёте.

При закрытии месяца все оплаты абонента за период суммируются и сохраняются в `paid_amount`.

При закрытии месяца все корректировки сальдо абонента за период суммируются и сохраняются в `adjustment_amount`.

Если у абонента есть предыдущее начисление, начальное сальдо берётся из `closing_balance` последнего предыдущего периода.

Если предыдущего начисления нет, начальное сальдо равно `0`.

Конечное сальдо считается так:

`closing_balance = opening_balance + amount - paid_amount + adjustment_amount`

Один абонент не может иметь два начисления за один расчётный месяц.

Повторное закрытие месяца в статусе `closed` запрещено.

Повторный запуск закрытия месяца в статусе `processing` запрещён.

Если закрытие завершилось с ошибками данных, расчётный месяц получает статус `failed`.

Месяц в статусе `failed` можно закрыть повторно после исправления данных.

Если хотя бы по одному активному абоненту есть ошибка данных, закрытие месяца не создаёт новые начисления ни по одному абоненту.

Ошибки закрытия сохраняются в отчёт расчётного месяца `billing_period_closure_errors`.

При повторном закрытии месяца старый отчёт ошибок очищается и заменяется новым результатом проверки.

Если по активному абоненту не хватает данных, начисление не создаётся, а процесс закрытия месяца возвращает ошибку с лицевым счётом, именем абонента и причиной.

Если для клиента с типом `per_person` нет действующего тарифа или `per_person_price`, начисление не создаётся, а закрытие месяца считает такую строку ошибкой данных.

Если для клиента с типом `meter` нет активных счётчиков, нет показания за период хотя бы по одному активному счётчику или нет действующего тарифа, начисление не создаётся, а закрытие месяца считает такую строку ошибкой данных.

## Административная панель

Начисления отображаются внутри карточки абонента через связанную таблицу начислений.

Отдельная страница начислений используется для общего контроля начислений и запуска закрытия текущего периода.

Закрытие месяца запускается кнопкой «Закрыть месяц» из списка начислений.

Система автоматически закрывает текущий расчётный месяц tenant-организации в статусе `open` или `failed`.

После запуска оператор получает уведомление с количеством активных абонентов, созданных начислений, пропущенных ранее созданных начислений и ошибок данных.

Ошибки показываются списком по лицевому счёту, абоненту и причине.

В связанной таблице начислений в карточке абонента должны отображаться период, услуга организации, тип начисления, объём, тариф, начисленная сумма, оплата, корректировка, начальное сальдо, конечное сальдо и дата закрытия.
