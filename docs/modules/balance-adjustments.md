# Модуль: Корректировки сальдо

## Терминология

Корректировка сальдо в коде называется `BalanceAdjustment`.

Таблица корректировок называется `balance_adjustments`.

## Назначение

Модуль хранит отдельные операции, которые изменяют сальдо абонента помимо начислений и оплат.

Корректировки используются для входящих остатков, ручных исправлений, списаний и перерасчётов.

## Основные поля

| Поле | Описание |
|---|---|
| Организация | Tenant-организация, поле `organization_id` |
| Абонент | Клиент, поле `client_id` |
| Расчётный месяц | Связь с `BillingPeriod`, поле `billing_period_id` |
| Тип | Тип корректировки, поле `type` |
| Сумма | Сумма корректировки, поле `amount` |
| Дата корректировки | Дата операции, поле `adjusted_at` |
| Причина / примечание | Дополнительная информация, поле `note` |

## Типы корректировок

Доступные значения `type`:

- `opening_balance` — входящий остаток;
- `manual_adjustment` — ручная корректировка;
- `write_off` — списание;
- `recalculation` — перерасчёт.

## Правила

Корректировка относится к tenant-организации, абоненту и расчётному периоду.

По одному абоненту может быть несколько корректировок за один расчётный месяц.

Положительная сумма увеличивает задолженность.

Отрицательная сумма уменьшает задолженность или увеличивает переплату.

При закрытии месяца все корректировки абонента за расчётный месяц суммируются.

При создании корректировки расчётный месяц не выбирается оператором.

Система автоматически привязывает корректировку к текущему расчётному месяцу tenant-организации в статусе `open` или `failed`.

Корректировку можно создать, изменить или удалить только в расчётном месяце со статусом `open` или `failed`.

В закрытом расчётном месяце корректировки менять нельзя.

Сумма корректировок за период сохраняется в начислении в поле `adjustment_amount`.

Конечное сальдо считается так:

`closing_balance = opening_balance + amount - paid_amount + adjustment_amount`

После закрытия месяца корректировки за этот расчётный месяц не изменяются.

## Административная панель

Корректировки сальдо управляются внутри `ClientResource` как связанная таблица конкретного абонента.

Отдельный раздел «Корректировки сальдо» не является основным рабочим сценарием.

Форма корректировки внутри карточки абонента должна позволять указать тип, сумму, дату корректировки и причину / примечание.

В связанной таблице корректировок должны отображаться период, тип, сумма, дата корректировки и дата создания.
