# Модуль: Расчётные месяцы

## Терминология

Расчётный месяц в коде называется `BillingPeriod`.

Таблица расчётных месяцев называется `billing_periods`.

Статус расчётного месяца хранится в enum `BillingPeriodStatus`.

## Назначение

Модуль хранит состояние расчётного месяца внутри tenant-организации.

Расчётный месяц является самостоятельной бизнес-сущностью, а не строковым полем в начислениях, оплатах или показаниях.

## Основные поля

| Поле | Описание |
|---|---|
| Организация | Tenant-организация, поле `organization_id` |
| Начало месяца | Первый день расчётного месяца, поле `starts_on` |
| Статус | Состояние месяца, поле `status` |
| Дата открытия | Когда месяц открыт, поле `opened_at` |
| Кто открыл | Пользователь, поле `opened_by_user_id` |
| Дата запуска закрытия | Когда началось закрытие, поле `processing_started_at` |
| Дата закрытия | Когда месяц успешно закрыт, поле `closed_at` |
| Кто закрыл | Пользователь, поле `closed_by_user_id` |
| Дата ошибки | Когда закрытие завершилось ошибками, поле `failed_at` |
| Причина ошибки | Общая причина ошибки закрытия, поле `failure_message` |
| Активных абонентов | Счётчик активных абонентов на момент закрытия, поле `active_clients_count` |
| Создано начислений | Счётчик созданных начислений, поле `created_accruals_count` |
| Пропущено начислений | Счётчик пропущенных существующих начислений, поле `skipped_accruals_count` |
| Ошибок данных | Счётчик абонентов с ошибками данных, поле `failed_clients_count` |

## Отчёт ошибок закрытия

Ошибки закрытия месяца хранятся в таблице `billing_period_closure_errors`.

Каждая строка ошибки относится к одному расчётному месяцу через `billing_period_id`.

Строка ошибки хранит tenant-организацию, абонента, лицевой счёт, имя абонента, тип начисления, стабильный код ошибки, текст причины и технический контекст.

Отчёт ошибок создаётся только при переходе расчётного месяца в статус `failed`.

При повторном запуске закрытия старый отчёт ошибок очищается и заменяется новым результатом проверки.

При успешном закрытии месяца отчёт ошибок очищается.

## Статусы

Доступные значения `status`:

- `open` — месяц открыт для ввода оплат, показаний и корректировок;
- `processing` — выполняется закрытие месяца;
- `closed` — месяц успешно закрыт и заблокирован для изменений;
- `failed` — закрытие завершилось ошибками данных, месяц можно исправить и закрыть повторно.

## Правила

Расчётный месяц уникален внутри tenant-организации по связке `organization_id + starts_on`.

`starts_on` всегда хранит первый день месяца.

Код месяца `ГГГГММ` является отображением `starts_on`, а не отдельной колонкой в учётных таблицах.

Первый расчётный месяц tenant-организации можно открыть без предыдущей истории.

Если у tenant-организации уже есть расчётные месяцы, новый месяц должен идти сразу после последнего существующего месяца.

Новый расчётный месяц нельзя создать, пока последний существующий расчётный месяц не закрыт.

Кнопка «Новый расчётный месяц» автоматически создаёт следующий календарный месяц после последнего закрытого месяца.

Оператор не вводит код месяца вручную при открытии нового расчётного месяца.

Начисления, оплаты, показания счётчиков, корректировки сальдо и квитанции ссылаются на расчётный месяц через `billing_period_id`.

Оплаты, показания счётчиков и корректировки сальдо можно создавать, изменять и удалять только для месяцев в статусах `open` и `failed`.

Для месяца в статусе `closed` учётные данные менять нельзя.

Повторное закрытие месяца в статусе `closed` запрещено.

Повторный запуск закрытия месяца в статусе `processing` запрещён.

Месяц в статусе `failed` можно закрыть повторно после исправления данных.

Закрытие месяца выполняется в транзакции и блокирует строку `billing_periods` через database row lock, чтобы два процесса не закрывали один месяц одновременно.

Если закрытие прошло без ошибок данных, месяц получает статус `closed`.

Если хотя бы по одному активному абоненту не хватает данных для расчёта, месяц получает статус `failed`.

Если месяц получил статус `failed`, оператор видит отчёт ошибок в разделе «Расчётные месяцы».

## Административная панель

Оператор управляет расчётными месяцами в разделе «Расчётные месяцы».

В разделе «Расчётные месяцы» кнопка «Новый расчётный месяц» открывает следующий месяц автоматически.

Закрытие месяца запускается кнопкой «Закрыть месяц» из раздела «Начисления».

Кнопка «Закрыть месяц» автоматически закрывает текущий расчётный месяц tenant-организации в статусе `open` или `failed`.

Для месяца с ошибками доступно действие «Отчёт ошибок», где отображаются лицевой счёт, абонент, тип начисления, причина и код ошибки.

В формах оплат, показаний и корректировок оператор не выбирает расчётный месяц.

Система автоматически привязывает оплаты, показания и корректировки к текущему расчётному месяцу tenant-организации в статусе `open` или `failed`.
