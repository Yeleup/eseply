# Модуль: Услуга клиента

## Назначение

Модуль фиксирует правило, что услуга клиента наследуется от выбранной tenant-организации.

Отдельный выбор услуги в карточке клиента не используется.

## Терминология

Абонент коммунальных услуг в коде называется `Client`.

Лицевой счёт является полем клиента, например `account_number`.

Услуга клиента — это услуга выбранной tenant-организации.

## Основные правила

У клиента не выбирается отдельная услуга.

Услуга клиента всегда равна услуге выбранной tenant-организации.

Внутри одной организации все клиенты обслуживаются по одной коммунальной услуге.

Услуга используется при:

- начислении;
- формировании квитанции;
- отчётах.

Если клиенту нужна другая коммунальная услуга, для этой услуги создаётся отдельная tenant-организация и клиент ведётся в ней отдельно.

История начислений сохраняет снимок услуги организации на момент расчёта.

## Связанные сущности

- Клиент
- Лицевой счёт
- Услуга организации (`UtilityService`)
- Тип начисления
- Категория тарифа (`TariffCategory`)
- Тариф
- Норматив
- Счётчик
- Начисление
- Квитанция

## Поля

| Поле | Описание |
|---|---|
| Клиент | Владелец услуги |
| Лицевой счёт | Основной лицевой счёт клиента |
| Услуга | Услуга выбранной tenant-организации, не выбирается в карточке клиента |
| Тип начисления | По счётчику / по нормативу / фиксированная сумма, поле `billing_type` |
| Категория тарифа | Категория `TariffCategory`, по которой определяется тариф, поле `tariff_category_id` |
| Количество проживающих | Используется при нормативном начислении, поле `residents_count` |
| Площадь | Используется при нормативном начислении, поле `area` |
| Фиксированная сумма | Используется только при фиксированном начислении, поле `fixed_amount` |
| Примечание | Дополнительная информация |

## Ограничения

В карточке клиента нельзя выбрать услугу.

Внутри одной организации нельзя подключить клиенту другую услугу.

Для новой услуги создаётся отдельная tenant-организация.

## UI

В карточке клиента нужно отображать:

- услугу выбранной tenant-организации;
- тип начисления;
- категорию тарифа;
- количество проживающих;
- площадь;
- фиксированную сумму, если выбран фиксированный тип;
- счётчик, если выбран тип “по счётчику”.
