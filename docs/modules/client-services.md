# Модуль: Услуга клиента

## Назначение

Модуль отвечает за выбор и хранение основной услуги клиента.

## Терминология

Абонент коммунальных услуг в коде называется `Client`.

Лицевой счёт является полем клиента, например `account_number`.

Услуга клиента — это выбранная основная услуга клиента.

## Основные правила

У каждого клиента выбирается одна основная услуга.

У одного клиента может быть только одна активная услуга.

Услуга выбирается в карточке клиента.

Услуга используется при:

- начислении;
- формировании квитанции;
- отчётах.

Если услугу нужно поменять, оператор может изменить её в карточке клиента.

История начислений по старой услуге должна сохраняться.

## Связанные сущности

- Клиент
- Лицевой счёт
- Услуга
- Тип начисления
- Категория тарифа
- Тариф
- Норматив
- Счётчик
- Начисление
- Квитанция

## Поля

| Поле | Описание |
|---|---|
| Клиент | Владелец услуги |
| Лицевой счёт | Основной лицевой счёт клиента |
| Услуга | Выбранная коммунальная услуга |
| Тип начисления | По счётчику / по нормативу / фиксированная сумма |
| Категория тарифа | Категория, по которой определяется тариф |
| Фиксированная сумма | Используется только при фиксированном начислении |
| Статус | Активная / неактивная |
| Дата начала | Когда услуга начала действовать |
| Дата окончания | Когда услуга была отключена |
| Примечание | Дополнительная информация |

## Ограничения

У клиента не может быть больше одной активной услуги.

При смене услуги старая услуга должна стать неактивной.

История старой услуги должна сохраняться.

## UI

В карточке клиента нужно отображать:

- текущую услугу;
- тип начисления;
- категорию тарифа;
- фиксированную сумму, если выбран фиксированный тип;
- счётчик, если выбран тип “по счётчику”.
