# Модуль: Клиенты

## Терминология

Абонент коммунальных услуг в коде называется `Client`.

Лицевой счёт является полем клиента, например `account_number`.

Услуга клиента — это выбранная основная услуга клиента из справочника `UtilityService`.

## Назначение

Модуль хранит карточки абонентов коммунальных услуг.

## Основные поля

| Поле | Описание |
|---|---|
| Организация | Tenant-организация клиента, поле `organization_id` |
| Лицевой счёт | Уникальный номер клиента, поле `account_number` |
| ФИО / Наименование | Имя физического лица или название организации |
| Тип клиента | Физическое лицо / юридическое лицо |
| Основная услуга | Коммунальная услуга клиента, поле `utility_service_id` |
| Категория тарифа | Категория для выбора тарифа, поле `tariff_category_id` |
| Тип начисления | По счётчику / по нормативу / фиксированная сумма, поле `billing_type` |
| Количество проживающих | Используется для расчёта по нормативу, поле `residents_count` |
| Площадь | Используется для расчёта по нормативу, поле `area` |
| Фиксированная сумма | Используется при фиксированном начислении, поле `fixed_amount` |
| Стартовое сальдо | Начальное сальдо клиента |
| Статус | Активный / неактивный |
| Телефон | Контактный номер |
| Адрес | Адрес клиента |
| Примечание | Дополнительная информация |

## Правила

Один клиент имеет один основной лицевой счёт.

Лицевой счёт должен быть уникальным внутри организации.

У клиента выбирается одна основная услуга из справочника `UtilityService`.

У клиента может быть выбрана категория тарифа из справочника `TariffCategory`.

Тип начисления хранится в карточке клиента в поле `billing_type`.

Доступные значения `billing_type`: `meter`, `normative`, `fixed`.

Параметры начисления первого этапа хранятся в таблице `clients`: `utility_service_id`, `tariff_category_id`, `billing_type`, `residents_count`, `area`, `fixed_amount`.

По лицевому счёту ведутся начисления, оплаты, сальдо, показания и квитанции.

## Административная панель

Абоненты управляются через Filament resource `ClientResource` внутри выбранной tenant-организации.

В навигации админки этот раздел называется «Абоненты» и находится в группе «Учёт» первым среди CRUD-разделов коммунального учёта.

В списке клиентов должны отображаться лицевой счёт, ФИО / наименование, тип клиента, услуга, категория тарифа, тип начисления, телефон, адрес, статус и стартовое сальдо.

Форма клиента должна позволять редактировать лицевой счёт, ФИО / наименование, тип клиента, услугу, категорию тарифа, тип начисления, количество проживающих, площадь, фиксированную сумму, телефон, адрес, статус, стартовое сальдо и примечание.
