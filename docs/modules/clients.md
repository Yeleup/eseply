# Модуль: Клиенты

## Терминология

Абонент коммунальных услуг в коде называется `Client`.

Лицевой счёт является полем клиента, например `account_number`.

Услуга клиента — это выбранная основная услуга клиента.

## Назначение

Модуль хранит карточки абонентов коммунальных услуг.

## Основные поля

| Поле | Описание |
|---|---|
| Лицевой счёт | Уникальный номер клиента, поле `account_number` |
| ФИО / Наименование | Имя физического лица или название организации |
| Тип клиента | Физическое лицо / юридическое лицо |
| Основная услуга | Выбранная услуга клиента |
| Тип начисления | По счётчику / по нормативу / фиксированная сумма |
| Категория тарифа | Категория, по которой определяется тариф |
| Количество проживающих | Используется для расчёта по нормативу |
| Площадь помещения | Используется для расчёта по нормативу, если нужно |
| Фиксированная сумма | Используется при фиксированном начислении |
| Стартовое сальдо | Начальное сальдо клиента |
| Статус | Активный / неактивный |
| Телефон | Контактный номер |
| Адрес | Адрес клиента |
| Примечание | Дополнительная информация |

## Правила

Один клиент имеет один основной лицевой счёт.

Лицевой счёт должен быть уникальным.

У клиента выбирается одна основная услуга.

Тип начисления хранится в карточке клиента.

По лицевому счёту ведутся начисления, оплаты, сальдо, показания и квитанции.
