# Модуль: Клиенты

## Терминология

Абонент коммунальных услуг в коде называется `Client`.

Лицевой счёт является полем клиента, например `account_number`.

Услуга клиента — это услуга выбранной tenant-организации.

## Назначение

Модуль хранит карточки абонентов коммунальных услуг.

## Основные поля

| Поле | Описание |
|---|---|
| Организация | Tenant-организация клиента, поле `organization_id` |
| Лицевой счёт | Уникальный номер клиента, поле `account_number` |
| ФИО / Наименование | Имя физического лица или название организации |
| ИИН | Обязательный ИИН клиента, уникальный внутри tenant-организации, поле `iin` |
| Тип клиента | Физ. лицо / ИП / ТОО / Коммерческие объекты / Бюджет, поле `client_type` |
| Услуга | Услуга выбранной tenant-организации, не выбирается в карточке клиента |
| Тип начисления | По счётчику / на одного человека / фиксированная сумма, поле `billing_type` |
| Количество проживающих | Обязательное поле клиента, по умолчанию `1`, используется для расчёта на одного человека, поле `residents_count` |
| Фиксированная сумма | Используется при фиксированном начислении, поле `fixed_amount` |
| Статус | Активный / неактивный |
| Телефон | Обязательный контактный номер, уникальный внутри tenant-организации |
| Договор | Обязательные реквизиты договора клиента, поле `contract` |
| Тех. условия | Технические условия клиента, поле `technical_conditions` |
| Регион | Регион из справочника tenant-организации, поле `region_id` |
| Улица | Улица выбранного региона, поле `street_id` |
| Дом | Номер дома, поле `house` |
| Квартира / помещение | Номер квартиры или помещения, поле `apartment` |
| Примечание | Дополнительная информация |

## Правила

Один клиент имеет один основной лицевой счёт.

Лицевой счёт должен быть уникальным внутри организации.

Лицевой счёт формируется автоматически при создании клиента.

Автоматическое формирование лицевого счёта ведётся отдельно внутри текущей tenant-организации.

Первый автоматически присвоенный лицевой счёт в каждой организации — `100001`; следующие клиенты получают следующий номер по порядку.

Оператор не редактирует лицевой счёт: поле отображается только для чтения.

После создания клиента лицевой счёт не меняется.

У клиента не выбирается отдельная услуга.

Услуга клиента всегда равна услуге выбранной tenant-организации.

Тип клиента хранится в поле `client_type` и выбирается из enum `ClientType`.

Доступные значения `ClientType`: Физ. лицо, ИП, ТОО, Коммерческие объекты, Бюджет.

Тип начисления хранится в карточке клиента в поле `billing_type`.

Доступные значения `billing_type`: `meter`, `per_person`, `fixed`.

Параметры начисления первого этапа хранятся в таблице `clients`: `client_type`, `billing_type`, `residents_count`, `fixed_amount`.

Поле `residents_count` является обязательным полем данных клиента, отображается в секции «Данные абонента», имеет значение по умолчанию `1` и не может быть меньше `1`.

Поля `iin`, `contract` и `phone` являются обязательными в форме клиента.

Поля `iin` и `phone` должны быть уникальными внутри tenant-организации. В разных tenant-организациях одинаковые ИИН и телефон допускаются.

Поле `technical_conditions` является nullable текстовым полем карточки клиента.

Поле `fixed_amount` показывается только при `billing_type = fixed`.

При `billing_type = meter` дополнительные поля настроек начисления не показываются.

Поле `utility_service_id` не должно быть пользовательской настройкой клиента.

Адрес клиента хранится структурно через поля `region_id`, `street_id`, `house`, `apartment`.

Поле `region_id` обязательно и выбирается из регионов текущей tenant-организации.

Поле `street_id` обязательно и выбирается только из улиц выбранного региона.

Выбранная улица должна принадлежать выбранному региону и текущей tenant-организации.

Поля `house` и `apartment` необязательные.

По лицевому счёту ведутся начисления, оплаты, сальдо, показания и квитанции.

Карточка клиента является основным рабочим экраном по лицевому счёту.

В карточке клиента должны отображаться связанные таблицы:

- счётчики клиента;
- оплаты клиента;
- корректировки сальдо клиента;
- начисления клиента;
- квитанции клиента.

Показания отображаются внутри карточки конкретного счётчика, который открыт из связанной таблицы счётчиков клиента.

## Административная панель

Абоненты управляются через Filament resource `ClientResource` внутри выбранной tenant-организации.

Для контроллера список абонентов и печатная карточка абонента ограничиваются назначенными регионами и улицами.

Контроллер может открыть страницу абонента из своей зоны ответственности только для просмотра контекста и перехода к счётчикам.

Контроллер не создаёт, не редактирует и не удаляет абонентов.

В навигации админки этот раздел называется «Абоненты» и находится в группе «Учёт» первым среди CRUD-разделов коммунального учёта.

В списке клиентов должны отображаться лицевой счёт, ФИО / наименование, тип клиента, услуга организации, тип начисления, телефон, регион, улица, дом, квартира / помещение и статус.

Форма клиента должна отображать лицевой счёт только для чтения, а также позволять редактировать ФИО / наименование, обязательные ИИН, телефон и договор, тип клиента, обязательное количество проживающих, тип начисления, фиксированную сумму для фиксированного начисления, тех. условия, регион, улицу, дом, квартиру / помещение, статус и примечание.

Услуга организации может отображаться в форме клиента только как справочная информация без выбора.

На странице редактирования клиента должна быть доступна кнопка «Карточка», которая открывает печатную Blade-карточку абонента в новой вкладке.

Печатная Blade-карточка абонента на текущем этапе содержит:

- все поля данных абонента, включая лицевой счёт, реквизиты, адрес, статус и настройки начисления;
- счётчики абонента;
- показания счётчиков;
- оплаты абонента;
- Kaspi-заявки абонента;
- корректировки сальдо;
- начисления;
- квитанции.

Печатная карточка использует компактную A4-разметку с табличными блоками и кнопкой «Печатать PDF», чтобы организации могли печатать карточку абонента с меньшим расходом бумаги.

Связанные таблицы в карточке клиента должны позволять:

- просматривать, добавлять и редактировать счётчики клиента;
- просматривать, добавлять и редактировать оплаты клиента;
- создавать Kaspi QR и просматривать историю Kaspi-заявок клиента;
- просматривать, добавлять и редактировать корректировки сальдо клиента;
- просматривать начисления клиента;
- просматривать квитанции клиента.

Для контроллера в карточке клиента отображается только связанная таблица счётчиков.

В связанной таблице счётчиков контроллер может добавить показание по счётчику из своей зоны ответственности.

Создание, редактирование, архивирование и удаление счётчиков в карточке клиента доступны только оператору.

Оплаты, корректировки сальдо, начисления и квитанции в карточке клиента доступны только оператору.
