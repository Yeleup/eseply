# Модуль: Адресные справочники

## Терминология

Регион в коде называется `Region`.

Улица в коде называется `Street`.

Таблица регионов называется `regions`.

Таблица улиц называется `streets`.

## Назначение

Модуль хранит справочники регионов и улиц внутри tenant-организации.

Регионы используются как верхний уровень адресного справочника.

Улицы принадлежат конкретному региону.

## Основные поля региона

| Поле | Описание |
|---|---|
| Организация | Tenant-организация, поле `organization_id` |
| Название | Название региона, поле `name` |

## Основные поля улицы

| Поле | Описание |
|---|---|
| Организация | Tenant-организация, поле `organization_id` |
| Регион | Регион, поле `region_id` |
| Название | Название улицы, поле `name` |

## Правила

Регион относится к tenant-организации.

Название региона должно быть уникальным внутри tenant-организации.

Улица относится к tenant-организации и региону.

`organization_id` улицы должен совпадать с `organization_id` региона.

Название улицы должно быть уникальным внутри одного региона tenant-организации.

При удалении организации удаляются её регионы и улицы.

При удалении региона удаляются его улицы.

В карточке абонента регион выбирается из справочника текущей tenant-организации.

После выбора региона список улиц должен показывать только улицы выбранного региона.

Нельзя сохранить абонента с улицей из другого региона или другой tenant-организации.

Регион или отдельная улица могут использоваться как зона ответственности контроллера внутри организации.

Назначение зоны ответственности контроллера действует только внутри конкретной привязки пользователя к организации.

Назначения регионов и улиц не используются для ролей, отличных от контроллера.

## Административная панель

Регионы управляются в профиле организации как связанный справочник.

Улицы управляются внутри карточки региона как связанный справочник.

Отдельный раздел регионов в навигации не используется.
