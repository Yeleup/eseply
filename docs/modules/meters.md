# Модуль: Счётчики

## Терминология

Счётчик в коде называется `Meter`.

Показание счётчика в коде называется `MeterReading`.

Таблица счётчиков называется `meters`.

Таблица показаний счётчиков называется `meter_readings`.

## Назначение

Модуль хранит счётчики абонентов и помесячные показания по ним.

Показания используются при закрытии месяца для клиентов с `billing_type = meter`.

## Основные поля счётчика

| Поле | Описание |
|---|---|
| Организация | Tenant-организация, поле `organization_id` |
| Абонент | Клиент, поле `client_id` |
| Услуга | Услуга выбранной tenant-организации, поле `utility_service_id` если хранится технически |
| Номер | Номер счётчика, поле `number` |
| Дата установки | Дата установки счётчика, поле `installed_on` |
| Начальное показание | Показание на момент установки, поле `initial_reading` |
| Дата снятия | Дата отправки счётчика в архив, поле `removed_on` |
| Статус | Активный / в архиве, поле `status` |
| Примечание | Дополнительная информация, поле `note` |

## Основные поля показания

| Поле | Описание |
|---|---|
| Организация | Tenant-организация, поле `organization_id` |
| Счётчик | Счётчик, поле `meter_id` |
| Абонент | Клиент, поле `client_id` |
| Услуга | Услуга выбранной tenant-организации, поле `utility_service_id` если хранится технически |
| Период | Расчётный месяц в формате `ГГГГММ`, поле `period` |
| Предыдущее показание | Значение на начало периода, поле `previous_reading` |
| Текущее показание | Значение на конец периода, поле `current_reading` |
| Расход | Разница текущего и предыдущего показания, поле `consumption` |
| Дата ввода | Дата внесения показания, поле `read_at` |
| Примечание | Дополнительная информация, поле `note` |

## Правила

Счётчик относится к tenant-организации и абоненту, а коммунальная услуга берётся из tenant-организации.

Номер счётчика уникален внутри tenant-организации.

У одного абонента может быть несколько активных счётчиков.

Каждый активный счётчик абонента участвует в расчёте по счётчику.

Снятые счётчики сохраняются для истории.

Дата снятия не редактируется вручную в форме счётчика.

Чтобы снять счётчик с активного учёта, оператор использует действие «Отправить в архив».

Действие «Отправить в архив» требует подтверждения, проставляет `removed_on` текущей датой и переводит `status` в `removed`.

Чтобы вернуть счётчик в активный учёт, оператор использует действие «Вывести из архива».

Действие «Вывести из архива» требует подтверждения, очищает `removed_on` и переводит `status` в `active`.

Одно показание может быть внесено только один раз по связке `meter_id + period`.

Одинаковый период может использоваться для разных счётчиков.

Основной ввод показаний выполняется внутри карточки счётчика через связанную таблицу `readings` или быстрым действием «Добавить показание» у счётчика в карточке абонента.

При создании показания поле `previous_reading` заполняется системой автоматически.

Для первого показания используется `meters.initial_reading`.

Для следующих периодов используется последнее `meter_readings.current_reading` этого счётчика за период раньше выбранного.

Расход считается по формуле:

`consumption = current_reading - previous_reading`

При сохранении показания `organization_id` и `client_id` берутся из выбранного счётчика.

Если `utility_service_id` хранится в счётчике или показании, он должен совпадать с услугой выбранной tenant-организации.

При закрытии месяца начисление по счётчику считается по формуле:

`amount = сумма consumption по всем активным счётчикам абонента * unit_price`

Если для активного клиента с `billing_type = meter` нет активных счётчиков по услуге организации или нет показания за период хотя бы по одному активному счётчику, начисление не создаётся и процесс закрытия месяца показывает ошибку.

## Административная панель

Счётчики управляются через Filament resource `MeterResource` внутри выбранной tenant-организации.

Основной список счётчиков абонента отображается в карточке клиента через связанную таблицу `meters`.

В связанной таблице `meters` у каждого счётчика доступно действие «Добавить показание», которое создаёт показание сразу для выбранного счётчика без перехода в карточку счётчика.

Показания управляются внутри `MeterResource` как связанная таблица показаний конкретного счётчика.

Отдельный раздел «Счётчики» не является основным рабочим сценарием; оператор открывает счётчик из карточки клиента.

Форма счётчика должна позволять выбрать абонента, номер, дату установки, начальное показание и примечание.

Дата снятия и статус архива управляются только действиями «Отправить в архив» и «Вывести из архива».

Форма показания внутри счётчика должна позволять указать период, текущее показание, дату ввода и примечание.

Предыдущее показание в форме показания должно быть уже заполнено и не должно требовать ручного ввода.

В связанной таблице показаний должны отображаться период, предыдущее показание, текущее показание, расход и дата ввода.
