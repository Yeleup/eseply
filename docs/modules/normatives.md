# Модуль: Нормативы

## Терминология

Норматив в коде называется `Normative`.

Таблица нормативов называется `normatives`.

## Назначение

Модуль хранит норму потребления услуги выбранной tenant-организации для категории тарифа и типа расчёта с даты начала действия.

## Основные поля

| Поле | Описание |
|---|---|
| Организация | Tenant-организация, поле `organization_id` |
| Услуга | Услуга выбранной tenant-организации, поле `utility_service_id` если хранится технически |
| Категория тарифа | Категория тарифа, поле `tariff_category_id` |
| Норма | Значение нормы, поле `value` |
| Тип расчёта | На человека / на объект / по площади, поле `calculation_type` |
| Дата начала | Дата начала действия норматива, поле `starts_on` |
| Статус | Активный / неактивный, поле `status` |

## Типы расчёта

- `per_person` — на человека;
- `per_object` — на объект;
- `per_area` — по площади.

## Правила

Каждая tenant-организация ведёт свои нормативы.

Норматив используется только для клиентов с типом начисления `normative`.

Норматив выбирается по услуге организации, категории тарифа, типу расчёта и расчётному периоду.

Если `utility_service_id` хранится у норматива, он должен совпадать с услугой выбранной tenant-организации.

Старые нормативы сохраняются для истории расчётов.

## Формулы начисления

На человека:

`Начисление = residents_count * normative * tariff`

По площади:

`Начисление = area * normative * tariff`

На объект:

`Начисление = normative * tariff`

В закрытии месяца используется активный норматив, действующий на начало расчётного периода.

## Административная панель

Нормативы управляются через Filament resource `NormativeResource` внутри выбранной tenant-организации.

В навигации админки раздел называется «Нормативы» и расположен после «Тарифы».

Форма норматива должна позволять редактировать категорию тарифа, норму, тип расчёта, дату начала и статус.

В списке нормативов должны отображаться услуга организации, категория тарифа, норма, тип расчёта, дата начала, статус и дата создания.
