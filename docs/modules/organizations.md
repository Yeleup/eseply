# Модуль организаций

## Назначение

Модуль хранит данные организации, от имени которой ведётся учёт коммунальных услуг.

В административной панели Filament организация используется как tenant-область.

Каждая tenant-организация ведёт учёт только по одной коммунальной услуге.

Если нужна новая услуга, создаётся отдельная tenant-организация.

## Терминология

Tenant-организация в коде называется `Organization`.

## Основные поля

- название организации;
- основная коммунальная услуга организации;
- БИН / ИИН;
- телефон;
- адрес;
- банк;
- IBAN;
- XPayment API key устройства;
- примечание.

## Правила доступа

- пользователь может быть участником нескольких организаций;
- пользователь может открыть только организацию, к которой он привязан;
- при создании организации через Filament текущий пользователь автоматически привязывается к созданной организации;
- роль пользователя хранится на привязке к организации;
- новый создатель организации получает роль оператора;
- доступные роли участника организации: оператор (`operator`) и контроллер (`controller`);
- оператор работает со всеми данными выбранной организации;
- контроллер может быть ограничен зоной ответственности по регионам и отдельным улицам организации;
- контроллер может просматривать абонентов и счётчики своей зоны ответственности как контекст для ввода показаний;
- контроллер может создавать и изменять только показания счётчиков в своей зоне ответственности;
- контроллер не управляет абонентами, счётчиками, оплатами, корректировками, начислениями, тарифами, квитанциями, адресными справочниками, профилем организации и пользователями;
- назначение регионов и улиц используется только для роли контроллера;
- при смене роли участника на роль, отличную от контроллера, назначения регионов и улиц очищаются;
- переключение между организациями выполняется через tenant menu Filament.

## Связи

- организация имеет много пользователей;
- организация имеет много клиентов;
- организация имеет одну основную коммунальную услугу;
- организация имеет много тарифов;
- организация имеет много регионов;
- организация имеет много улиц через регионы;
- организация имеет много назначений регионов и улиц для контроллеров через привязку пользователя к организации;
- пользователь имеет много организаций.

## Ограничения

Тип начисления не относится к организации и хранится только в карточке абонента.

Внутри одной организации нельзя добавить вторую коммунальную услугу.

Абоненты, тарифы, счётчики, начисления и квитанции используют услугу выбранной организации.

XPayment API key уникален для каждой tenant-организации и хранится в зашифрованном виде.

Глобальные `.env` настройки XPayment используются для технических параметров клиента и общего webhook secret: base URL, webhook secret, timeout и connect timeout.

## Профиль организации

Профиль tenant-организации доступен только оператору выбранной организации.

Оператор может настроить XPayment API key устройства в профиле tenant-организации.

Пользователи организации управляются в профиле tenant-организации как связанный справочник.

В профиле организации можно прикрепить существующего пользователя, создать нового пользователя, указать роль участника и для контроллера назначить регионы и отдельные улицы.

Регионы организации управляются в профиле tenant-организации как связанный справочник.

Улицы управляются внутри карточки региона.
