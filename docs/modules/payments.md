# Модуль: Оплаты

## Терминология

Оплата в коде называется `Payment`.

Таблица оплат называется `payments`.

## Назначение

Модуль хранит оплаты абонентов по расчётным периодам.

Оплаты используются при закрытии месяца для расчёта конечного сальдо.

## Основные поля

| Поле | Описание |
|---|---|
| Организация | Tenant-организация, поле `organization_id` |
| Абонент | Клиент, поле `client_id` |
| Расчётный месяц | Связь с `BillingPeriod`, поле `billing_period_id` |
| Сумма | Сумма оплаты, поле `amount` |
| Дата оплаты | Дата поступления оплаты, поле `paid_at` |
| Примечание | Дополнительная информация, поле `note` |

## Правила

Оплата относится к tenant-организации и абоненту.

По одному абоненту может быть несколько оплат за один расчётный месяц.

При закрытии месяца все оплаты абонента за расчётный месяц суммируются.

При создании оплаты расчётный месяц не выбирается оператором.

Система автоматически привязывает оплату к текущему расчётному месяцу tenant-организации в статусе `open` или `failed`.

Оплату можно создать, изменить или удалить только в расчётном месяце со статусом `open` или `failed`.

В закрытом расчётном месяце оплаты менять нельзя.

Если за текущий расчётный месяц у абонента уже есть квитанция, создание, изменение или удаление оплаты обновляет в ней `paid_amount` и `closing_balance`.

Оплата не создаёт квитанцию сама; квитанция создаётся при сохранении показания счётчика.

Сумма оплат за период сохраняется в начислении в поле `paid_amount`.

Конечное сальдо считается так:

`closing_balance = opening_balance + amount - paid_amount + adjustment_amount`

## Административная панель

Оплаты управляются внутри `ClientResource` как связанная таблица оплат конкретного абонента.

Отдельный раздел «Оплаты» не является основным рабочим сценарием.

Форма оплаты внутри карточки абонента должна позволять указать сумму, дату оплаты и примечание.

В связанной таблице оплат должны отображаться период, сумма, дата оплаты и дата создания.
