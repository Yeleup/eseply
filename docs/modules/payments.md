# Модуль: Оплаты

## Терминология

Оплата в коде называется `Payment`.

Таблица оплат называется `payments`.

## Назначение

Модуль хранит оплаты абонентов по расчётным периодам.

Оплаты используются при закрытии месяца для расчёта конечного сальдо.

## Основные поля

| Поле | Описание |
|---|---|
| Организация | Tenant-организация, поле `organization_id` |
| Абонент | Клиент, поле `client_id` |
| Период | Расчётный месяц в формате `ГГГГММ`, поле `period` |
| Сумма | Сумма оплаты, поле `amount` |
| Дата оплаты | Дата поступления оплаты, поле `paid_at` |
| Примечание | Дополнительная информация, поле `note` |

## Правила

Оплата относится к tenant-организации и абоненту.

По одному абоненту может быть несколько оплат за один период.

При закрытии месяца все оплаты абонента за период суммируются.

Сумма оплат за период сохраняется в начислении в поле `paid_amount`.

Конечное сальдо считается так:

`closing_balance = opening_balance + amount - paid_amount`

## Административная панель

Оплаты управляются через Filament resource `PaymentResource` внутри выбранной tenant-организации.

В навигации админки раздел называется «Оплаты».

Форма оплаты должна позволять выбрать абонента, период, сумму, дату оплаты и примечание.

В списке оплат должны отображаться период, лицевой счёт, абонент, сумма, дата оплаты и дата создания.
