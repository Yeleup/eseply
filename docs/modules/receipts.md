# Модуль: Квитанции

## Терминология

Квитанция в коде называется `Receipt`.

Таблица квитанций называется `receipts`.

## Назначение

Модуль хранит сформированные квитанции за закрытый расчётный месяц.

Квитанция создаётся после закрытия месяца из сохранённого `Accrual`.

Квитанция не пересчитывает начисление, оплату или сальдо на лету.

## Основные поля

| Поле | Описание |
|---|---|
| Организация | Tenant-организация, поле `organization_id` |
| Абонент | Клиент, поле `client_id` |
| Начисление | Источник квитанции, поле `accrual_id` |
| Номер квитанции | Уникальный номер внутри организации, поле `receipt_number` |
| Расчётный месяц | Связь с `BillingPeriod`, поле `billing_period_id` |
| Лицевой счёт | Снимок лицевого счёта из начисления, поле `account_number` |
| Абонент | Снимок имени / наименования из начисления, поле `client_name` |
| Название услуги | Снимок услуги организации из начисления, поле `utility_service_name` |
| Тип начисления | Тип начисления из начисления, поле `billing_type` |
| Объём | Сохранённый объём из начисления, поле `volume` |
| Тариф | Сохранённый тариф из начисления, поле `tariff_price` |
| Начислено | Сохранённая сумма начисления, поле `amount` |
| Оплачено | Сохранённая сумма оплат, поле `paid_amount` |
| Корректировка | Сохранённая сумма корректировок сальдо, поле `adjustment_amount` |
| Начальное сальдо | Сохранённое начальное сальдо, поле `opening_balance` |
| Конечное сальдо | Сохранённое конечное сальдо, поле `closing_balance` |
| Дата формирования | Когда создана квитанция, поле `issued_at` |

## Правила

Квитанция формируется по одному абоненту за один расчётный период.

Один абонент не может иметь две квитанции за один расчётный месяц.

Один `Accrual` не может иметь больше одной квитанции.

Номер квитанции формируется как `код расчётного месяца-account_number`, например `202605-100001`.

Квитанция создаётся автоматически при успешном создании начисления во время закрытия месяца.

Повторное закрытие закрытого расчётного месяца запрещено.

Квитанция строится только из полей `Accrual`; изменение карточки клиента, тарифа, оплаты или счётчика после закрытия не меняет уже сформированную квитанцию.

## Административная панель

Квитанции отображаются внутри карточки абонента через связанную таблицу квитанций.

Отдельный раздел «Квитанции» не является основным рабочим сценарием.

Оператор может открыть карточку абонента, увидеть его квитанции, фильтровать их по периоду и просматривать квитанцию.

На странице просмотра квитанции оператор может нажать «Печатать PDF», чтобы открыть печатное представление квитанции в новой вкладке и сохранить или распечатать его через браузерный PDF-диалог.

Печатное представление использует сохранённые поля `Receipt` для расчёта сумм и не пересчитывает начисления, оплаты или сальдо.

Ручное создание и редактирование квитанций в первом этапе не используется.
