# Модуль: Квитанции

## Терминология

Квитанция в коде называется `Receipt`.

Таблица квитанций называется `receipts`.

## Назначение

Модуль хранит квитанции абонентов за расчётный месяц.

Квитанция создаётся автоматически при сохранении показания счётчика `MeterReading`.

Если показание счётчика за тот же расчётный месяц изменяется, существующая квитанция этого абонента и периода обновляется.

Квитанция не хранит обязательную информацию о начислении клиента и не создаётся закрытием месяца из `Accrual`.

## Основные поля

| Поле | Описание |
|---|---|
| Организация | Tenant-организация, поле `organization_id` |
| Абонент | Клиент, поле `client_id` |
| Расчётный месяц | Связь с `BillingPeriod`, поле `billing_period_id` |
| Начисление | Необязательная техническая ссылка для старых данных, поле `accrual_id` |
| Номер квитанции | Уникальный номер внутри организации, поле `receipt_number` |
| Лицевой счёт | Снимок лицевого счёта абонента, поле `account_number` |
| Абонент | Снимок имени / наименования абонента, поле `client_name` |
| Название услуги | Снимок услуги организации, поле `utility_service_name` |
| Тип расчёта | Тип расчёта абонента, поле `billing_type` |
| Объём | Суммарный расход по показаниям счётчиков абонента за период, поле `volume` |
| Тариф | Использованная цена за единицу, поле `tariff_price` |
| Сумма | Сумма к оплате по показаниям и тарифу, поле `amount` |
| Список счётчиков | Строки показаний счётчиков абонента за период: номер счётчика, предыдущее показание, текущее показание, расход, тариф и сумма |
| Оплачено | Сохранённая сумма оплат, поле `paid_amount` |
| Корректировка | Сохранённая сумма корректировок сальдо, поле `adjustment_amount` |
| Начальное сальдо | Сохранённое начальное сальдо, поле `opening_balance` |
| Конечное сальдо | Сохранённое конечное сальдо, поле `closing_balance` |
| Дата формирования | Когда квитанция была сформирована или обновлена, поле `issued_at` |

## Правила

Квитанция формируется по одному абоненту за один расчётный период.

Один абонент не может иметь две квитанции за один расчётный месяц.

Номер квитанции формируется как `код расчётного месяца-account_number`, например `202605-100001`.

При создании показания счётчика система создаёт квитанцию для абонента и расчётного месяца показания.

При изменении показания счётчика система обновляет ту же квитанцию, а не создаёт новую.

Если за текущий расчётный месяц у абонента уже есть квитанция, создание, изменение или удаление оплаты обновляет в ней сумму оплат и конечное сальдо.

Если квитанции за расчётный месяц ещё нет, оплата не создаёт её.

Если за период у абонента есть несколько счётчиков, квитанция хранит суммарный расход по всем показаниям абонента за этот период.

В просмотре и печатной форме квитанция должна показывать список счётчиков с показаниями, расходом, тарифом и суммой по каждой строке.

Закрытие месяца создаёт `Accrual`, но не создаёт и не обновляет `Receipt`.

Повторное закрытие закрытого расчётного месяца запрещено.

## Административная панель

Квитанции отображаются внутри карточки абонента через связанную таблицу квитанций.

Отдельный раздел «Квитанции» не является основным рабочим сценарием.

Оператор может открыть карточку абонента, увидеть его квитанции, фильтровать их по периоду и просматривать квитанцию.

На странице просмотра квитанции оператор может нажать «Печатать PDF», чтобы открыть печатное представление квитанции в новой вкладке и сохранить или распечатать его через браузерный PDF-диалог.

Печатное представление использует сохранённые поля `Receipt` для вывода сумм и не показывает обязательную информацию о начислении клиента.

Ручное создание и редактирование квитанций в первом этапе не используется.
