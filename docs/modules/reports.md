# Модуль: Отчёты

## Назначение

Модуль предоставляет отчёты по сохранённым данным выбранной tenant-организации.

Отчёты открываются отдельным разделом Filament «Отчёты» и не размещаются внутри карточки абонента.

## Архитектура

Один отчёт реализуется отдельным классом в `app/Reports`.

Доступные отчёты регистрируются в `App\Reports\ReportRegistry`.

Filament-страницы отвечают за навигацию и отображение, а запросы и колонки отчёта находятся в классе конкретного отчёта.

## Ведомость снятия показаний

Ведомость снятия показаний в коде называется `MeterReadingSheetReport`.

Отчёт показывает активные счётчики активных абонентов выбранной tenant-организации.

Для контроллера отчёт показывает только активные счётчики активных абонентов из назначенных регионов и улиц.

Строки сортируются по лицевому счёту абонента и номеру счётчика.

Если у одного абонента несколько счётчиков, они отображаются соседними строками.

Колонки отчёта:

- лицевой счёт;
- ФИО / наименование абонента;
- адрес;
- количество проживающих;
- счётчик;
- дата установки счётчика;
- предыдущее показание;
- пустая колонка для нового показания.

Предыдущее показание берётся из последнего сохранённого текущего показания счётчика.

Если по счётчику ещё нет сохранённых показаний, предыдущее показание равно начальному показанию счётчика.

Ведомость можно скачать со страницы отчёта в формате XLSX.

XLSX-выгрузка использует тот же tenant-scope, тот же порядок строк и те же колонки, что и экранный отчёт.

Для контроллера XLSX-выгрузка использует тот же фильтр зоны ответственности, что и экранная таблица.

## Список не снятых показаний

Список не снятых показаний в коде называется `MissingMeterReadingsReport`.

Отчёт показывает активные счётчики активных абонентов выбранной tenant-организации с типом начисления `meter`, по которым нет показания за текущий открытый или ошибочный расчётный месяц.

Если текущий расчётный месяц не открыт, отчёт не показывает строки и сообщает, что месяц нужно открыть.

Для контроллера отчёт показывает только активные счётчики активных абонентов из назначенных регионов и улиц.

Строки сортируются по лицевому счёту абонента и номеру счётчика.

Колонки отчёта:

- лицевой счёт;
- ФИО / наименование абонента;
- адрес;
- количество проживающих;
- счётчик;
- дата установки счётчика;
- текущий расчётный период;
- предыдущее показание.

Предыдущее показание берётся из последнего сохранённого текущего показания счётчика до текущего расчётного месяца.

Если по счётчику ещё нет сохранённых показаний, предыдущее показание равно начальному показанию счётчика.

Список можно скачать со страницы отчёта в формате XLSX.

XLSX-выгрузка использует тот же tenant-scope, тот же период, тот же порядок строк и те же колонки, что и экранный отчёт.

## Процент снятия по контроллерам

Процент снятия по контроллерам в коде называется `ControllerMeterReadingProgressReport`.

Отчёт показывает пользователей выбранной tenant-организации с ролью `controller`.

Отчёт строится за текущий открытый или ошибочный расчётный месяц.

Если текущий расчётный месяц не открыт, отчёт не показывает строки и сообщает, что месяц нужно открыть.

Для каждого контроллера система определяет зону ответственности по назначенным регионам и улицам.

В знаменатель процента входят активные счётчики активных абонентов с типом начисления `meter`, которые попадают в зону ответственности контроллера.

В числитель процента входят счётчики из знаменателя, по которым есть показание за текущий расчётный месяц.

Если счётчик попадает к контроллеру одновременно по региону и улице, он учитывается один раз.

Если у контроллера нет назначенной зоны или в зоне нет активных счётчиков, общее количество, снятые показания, не снятые показания и процент равны нулю.

Колонки отчёта:

- контроллер;
- email;
- назначенные регионы;
- назначенные улицы;
- текущий расчётный период;
- всего счётчиков;
- снято;
- не снято;
- процент снятия.

Отчёт можно скачать со страницы отчёта в формате XLSX.

XLSX-выгрузка использует тот же tenant-scope, тот же период, тот же порядок контроллеров и те же колонки, что и экранный отчёт.
