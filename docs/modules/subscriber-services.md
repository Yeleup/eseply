# Модуль: Услуга абонента

## Назначение

Модуль отвечает за выбор и хранение основной услуги абонента.

## Основные правила

У каждого абонента выбирается одна основная услуга.

У одного абонента может быть только одна активная услуга.

Услуга выбирается в карточке абонента.

Услуга используется при:

- начислении;
- формировании квитанции;
- отчётах.

Если услугу нужно поменять, оператор может изменить её в карточке абонента.

История начислений по старой услуге должна сохраняться.

## Связанные сущности

- Абонент
- Лицевой счёт
- Услуга
- Тип начисления
- Категория тарифа
- Тариф
- Норматив
- Счётчик
- Начисление
- Квитанция

## Поля

| Поле | Описание |
|---|---|
| Абонент | Владелец услуги |
| Лицевой счёт | Основной лицевой счёт абонента |
| Услуга | Выбранная коммунальная услуга |
| Тип начисления | По счётчику / по нормативу / фиксированная сумма |
| Категория тарифа | Категория, по которой определяется тариф |
| Фиксированная сумма | Используется только при фиксированном начислении |
| Статус | Активная / неактивная |
| Дата начала | Когда услуга начала действовать |
| Дата окончания | Когда услуга была отключена |
| Примечание | Дополнительная информация |

## Ограничения

У абонента не может быть больше одной активной услуги.

При смене услуги старая услуга должна стать неактивной.

История старой услуги должна сохраняться.

## UI

В карточке абонента нужно отображать:

- текущую услугу;
- тип начисления;
- категорию тарифа;
- фиксированную сумму, если выбран фиксированный тип;
- счётчик, если выбран тип “по счётчику”.
