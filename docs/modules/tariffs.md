# Модуль: Тарифы

## Терминология

Тариф в коде называется `Tariff`.

Таблица тарифов называется `tariffs`.

## Назначение

Модуль хранит цену услуги выбранной tenant-организации для категории тарифа с даты начала действия.

## Основные поля

| Поле | Описание |
|---|---|
| Организация | Tenant-организация, поле `organization_id` |
| Услуга | Услуга выбранной tenant-организации, поле `utility_service_id` если хранится технически |
| Категория тарифа | Категория тарифа, поле `tariff_category_id` |
| Цена | Цена за единицу, поле `price` |
| Дата начала | Дата начала действия тарифа, поле `starts_on` |
| Статус | Активный / неактивный, поле `status` |

## Правила

Каждая tenant-организация ведёт свои тарифы.

Тариф выбирается по услуге организации, категории тарифа и расчётному периоду.

Если `utility_service_id` хранится у тарифа, он должен совпадать с услугой выбранной tenant-организации.

В расчёте используется тариф, действующий на выбранный период.

Старые тарифы сохраняются для истории расчётов.

## Административная панель

Тарифы управляются через Filament resource `TariffResource` внутри выбранной tenant-организации.

В навигации админки раздел называется «Тарифы» и расположен после «Категории тарифов».

Форма тарифа должна позволять редактировать категорию тарифа, цену, дату начала и статус.

В списке тарифов должны отображаться услуга организации, категория тарифа, цена, дата начала, статус и дата создания.
