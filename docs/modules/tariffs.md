# Модуль: Тарифы

## Терминология

Тариф в коде называется `Tariff`.

Таблица тарифов называется `tariffs`.

## Назначение

Модуль хранит цены услуги выбранной tenant-организации для типа абонента с даты начала действия.

## Основные поля

| Поле | Описание |
|---|---|
| Организация | Tenant-организация, поле `organization_id` |
| Услуга | Услуга выбранной tenant-организации, поле `utility_service_id` если хранится технически |
| Тип абонента | Тип абонента, поле `client_type` |
| Цена за единицу | Цена за 1 м³, 1 кВт⋅ч или другую единицу услуги, поле `unit_price` |
| Цена на одного человека | Сумма на одного проживающего, поле `per_person_price` |
| Дата начала | Дата начала действия тарифа, поле `starts_on` |
| Статус | Активный / неактивный, поле `status` |

## Правила

Каждая tenant-организация ведёт свои тарифы.

Тариф выбирается по услуге организации, типу абонента и расчётному периоду.

Если `utility_service_id` хранится у тарифа, он должен совпадать с услугой выбранной tenant-организации.

В расчёте используется тариф, действующий на выбранный период.

Для начисления по счётчику используется `unit_price`.

Для начисления на одного человека используется `per_person_price`.

Старые тарифы сохраняются для истории расчётов.

## Административная панель

Тарифы управляются через Filament resource `TariffResource` внутри выбранной tenant-организации.

В навигации админки раздел называется «Тарифы» и расположен после «Абоненты».

Форма тарифа должна позволять редактировать тип абонента, цену за единицу, цену на одного человека, дату начала и статус.

В списке тарифов должны отображаться услуга организации, тип абонента, цена за единицу, цена на одного человека, дата начала, статус и дата создания.
