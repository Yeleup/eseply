# Модуль: Услуга организации

## Терминология

Коммунальная услуга организации технически может храниться через `UtilityService`.

Таблица справочных значений услуг называется `utility_services`.

Название `Service` не используется для модели, чтобы не путать справочник коммунальных услуг с сервисными классами Laravel.

## Назначение

Модуль хранит коммунальную услугу, под которую создана tenant-организация.

Внутри tenant-организации не ведётся набор услуг.

Если нужно вести новую услугу, создаётся отдельная tenant-организация.

## Основные поля

| Поле | Описание |
|---|---|
| Организация | Tenant-организация, для которой задана услуга |
| Название | Название коммунальной услуги, поле `name` |
| Единица измерения | Единица измерения услуги, поле `unit_of_measurement` |
| Статус | Активная / неактивная, поле `status` |
| Примечание | Дополнительная информация, поле `note` |

## Правила

Каждая tenant-организация имеет ровно одну основную коммунальную услугу.

Внутри одной организации нельзя добавить вторую коммунальную услугу.

Услуга не выбирается в карточке клиента.

Услуга клиента всегда равна услуге выбранной tenant-организации.

Если нужна другая коммунальная услуга, создаётся отдельная tenant-организация.

Изменение услуги организации допускается только до начала учёта или как административное исправление.

## Связи

Коммунальная услуга используется в тарифах `Tariff`.

Коммунальная услуга сохраняется снимком в начислениях и квитанциях.

## Административная панель

Услуга задаётся при создании или настройке tenant-организации.

Отдельный tenant-раздел для добавления нескольких услуг внутри организации не используется.

Форма организации должна показывать услугу организации и единицу измерения.

Оператор не должен выбирать услугу при создании или редактировании клиента.
