# Модуль: Справочник коммунальных услуг

## Терминология

Справочник коммунальных услуг в коде называется `UtilityService`.

Таблица справочника услуг называется `utility_services`.

Название `Service` не используется для модели, чтобы не путать справочник коммунальных услуг с сервисными классами Laravel.

## Назначение

Модуль хранит перечень коммунальных услуг, которые tenant-организация использует для клиентов, тарифов, нормативов, начислений, квитанций и отчётов.

## Основные поля

| Поле | Описание |
|---|---|
| Организация | Tenant-организация, поле `organization_id` |
| Название | Название коммунальной услуги, поле `name` |
| Единица измерения | Единица измерения услуги, поле `unit_of_measurement` |
| Статус | Активная / неактивная, поле `status` |
| Примечание | Дополнительная информация, поле `note` |

## Правила

Каждая tenant-организация ведёт свой справочник коммунальных услуг.

Услуги одной организации не должны быть доступны в другой организации.

Активные услуги доступны для выбора в карточке клиента.

Неактивные услуги сохраняются для истории и не должны удалять историю начислений.

## Связи

Коммунальная услуга используется в тарифах `Tariff`.

Коммунальная услуга используется в нормативах `Normative`.

## Административная панель

Коммунальные услуги управляются через Filament resource `UtilityServiceResource` внутри выбранной tenant-организации.

В навигации админки раздел называется «Услуги» и расположен после «Абоненты».

Форма услуги должна позволять редактировать название, единицу измерения, статус и примечание.

В списке услуг должны отображаться название, единица измерения, статус и дата создания.
